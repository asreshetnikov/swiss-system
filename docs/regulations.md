# Регламент — SwissChess

Настоящий документ описывает алгоритм жеребьёвки, правила определения цвета, порядок расчёта турнирной таблицы и коэффициента Бухгольца, применяемые в системе SwissChess.

Алгоритм основан на **Нидерландской системе (Dutch System)** швейцарской жеребьёвки. Нормативный источник: **FIDE Handbook, раздел C.04** — *Dutch System of the Swiss Tournament* (https://handbook.fide.com/chapter/C04Annex1).

## 1. Посевной номер (seed)

Посевные номера присваиваются в момент перевода турнира из статуса **OPEN** в **RUNNING**:

- участники сортируются по рейтингу **по убыванию**;
- при равном рейтинге — по порядку добавления в турнир (**по возрастанию** id записи);
- первому в этом порядке присваивается seed = 1, далее по нарастанию.

Посевной номер фиксируется на всё время турнира и используется в жеребьёвке как **мера ранга**: меньший номер — более высокий ранг.

> FIDE Dutch System, ст. 1: «Players are ranked by their score, and within the same score by their starting rank.»

## 2. Жеребьёвка первого тура

### 2.1 Разбивка на половины

Все активные участники сортируются по посевному номеру (по возрастанию). Список делится ровно пополам:

- **верхняя половина** — участники с seed 1 … N/2;
- **нижняя половина** — участники с seed N/2+1 … N.

Пара формируется из участника верхней половины и соответствующего ему (по позиции) участника нижней половины:

```
Board 1:  seed 1   vs  seed (N/2 + 1)
Board 2:  seed 2   vs  seed (N/2 + 2)
…
```

> FIDE Dutch System, ст. 4 (Round 1): «The players are ranked according to their rating. The first half plays the second half.»

### 2.2 Назначение цвета

- **Цвет первой доски** определяется случайным образом (жребий) — с равной вероятностью белые получает участник верхней или нижней половины.
- Для каждой следующей доски цвет **чередуется**: если на доске №1 верхняя половина играет белыми, то на доске №2 нижняя половина играет белыми, на доске №3 — снова верхняя, и т. д.

> FIDE Dutch System, ст. 4: «Colors are assigned by lot for the first game; thereafter colors alternate in the pairing order.»

### 2.3 Нечётное число участников

Если участников нечётное число, перед формированием пар выбирается получатель BYE (см. раздел 5), после чего оставшиеся чётное число участников делятся на половины.

## 3. Жеребьёвка последующих туров

### 3.1 Формирование очковых групп

Все активные участники разбиваются на **очковые группы** — группы по одинаковому числу набранных очков, отсортированные по убыванию очков.

> FIDE Dutch System, ст. 5: «Players are grouped by their score.»

### 3.2 Порядок участников внутри группы

Перед попыткой составить пары участники внутри каждой группы сортируются:

1. **|CD| по убыванию** — участник с большим дисбалансом цветов обрабатывается первым, чтобы он мог получить приоритет при выборе партнёра с дополнительным цветом.
2. **Seed по возрастанию** — при одинаковом |CD| более высокорейтинговый участник стоит выше.

Здесь CD (Color Difference) — разница между числом сыгранных партий белыми и чёрными (подробнее — в разделе 4).

### 3.3 Двухпроходной алгоритм составления пар

Для каждого участника (берётся первый в отсортированном списке) последовательно выполняются два прохода:

**Проход 1 — идеальная пара:**
Ищется первый возможный соперник, для которого одновременно выполняются оба условия:
1. дополнительный причитающийся цвет (complementary color due) — один игрок должен получить белые, другой чёрные;
2. отсутствие предыдущей встречи между этими участниками.

**Проход 2 — принятие цветового нарушения:**
Если идеальный соперник не найден, ищется любой соперник без предыдущей встречи (цветовое предпочтение игнорируется).

**Флоутер:**
Если для участника не удалось найти никакого соперника без повтора, участник «опускается» (floats down) в следующую очковую группу.

> FIDE Dutch System, ст. 5: «Pairing within score bracket: first try to pair with complementary color; if not possible, accept color violation rather than repeat opponent.»

### 3.4 Выбор флоутера (нечётная группа)

Если количество игроков в очковой группе нечётное (с учётом уже пришедших сверху флоутеров), один игрок должен быть опущен в следующую группу.

Выбор флоутера производится заранее, до попытки составить пары в группе, с учётом **цветовых потребностей следующей группы**:

1. Определяется, каких цветов «должна» большинство игроков следующей группы.
2. Из кандидатов на опускание (только «родные» игроки группы, уже пришедшие сверху флоутеры не могут быть опущены повторно) выбирается тот, чей причитающийся цвет дополняет доминирующую потребность следующей группы.
3. Если подходящего кандидата нет или следующая группа сбалансирована по цветам — опускается наихудший по рангу игрок группы.

> FIDE Dutch System, ст. 5: «The floater should be chosen so as to minimize color conflicts in the receiving score bracket.»

### 3.5 Принудительные повторы

Если после обработки всех групп остаются несопоставленные игроки (не удалось избежать повторных встреч ни в одной комбинации), такие участники принудительно объединяются в пары с нарушением правила о повторах. Это ситуация последнего резерва и в нормально проходящих турнирах не должна возникать.

## 4. Определение цвета в паре

### 4.1 Разница цветов (CD)

Для каждого участника отслеживается **Color Difference (CD)**:

```
CD = (количество партий белыми) − (количество партий чёрными)
```

- CD > 0 → участник играл белыми чаще, следующая партия «причитается» чёрными.
- CD < 0 → участник играл чёрными чаще, следующая партия «причитается» белыми.
- CD = 0 → участник должен играть цветом, противоположным цвету его последней партии. При отсутствии истории — предпочтение белых.

> FIDE Dutch System, ст. 5: «Color preference is absolute when |CD| ≥ 2; otherwise it is a mild obligation.»

### 4.2 Назначение цвета в паре

Пусть p1 — игрок с более высоким рангом (меньший seed / больше очков), p2 — второй игрок.

**Случай 1: дополнительные причитающиеся цвета (complementary)**
Причитающиеся цвета p1 и p2 различны (один должен получить белые, другой — чёрные). Каждый игрок получает свой причитающийся цвет.

**Случай 2: конфликт (оба претендуют на один цвет)**
Приоритет получает игрок с **большим |CD|** — у него дисбаланс выражен сильнее.

При равном |CD| приоритет получает **p1** (более высокоранговый игрок).

Игрок с приоритетом получает свой причитающийся цвет; второй играет противоположным.

> FIDE Dutch System, ст. 5: «If there is a color conflict, the player with the stronger color preference (higher |CD|) receives the preferred color.»

## 5. Технический выигрыш (BYE)

### 5.1 Назначение BYE

Если число активных участников нечётное, один участник получает технический выигрыш (BYE) и в туре не играет.

Получатель BYE определяется по **позиции в текущей турнирной таблице** — снизу:

1. Сортировка активных участников: **меньше очков — ниже; при равных очках — больший seed — ниже**.
2. Из участников, ещё не получавших BYE, выбирается последний в этом порядке.
3. Если все участники уже получали BYE, ограничение снимается и снова берётся последний.

> FIDE Dutch System, ст. 5: «The bye is assigned to the lowest-ranked player in the lowest score bracket who has not yet received a bye.»

### 5.2 Очки за BYE

Количество очков, начисляемых за BYE, задаётся при создании турнира (поле **Bye points**, обычно 1 очко).

## 6. Турнирная таблица и тайбрейки

Итоговый (и промежуточный) порядок участников определяется последовательным применением следующих критериев:

| Приоритет | Критерий | Направление |
|-----------|----------|-------------|
| 1 | Очки | по убыванию |
| 2–4 | Настраиваемые тайбрейки | см. ниже |
| (финальный) | Посевной номер | по возрастанию |

### 6.1 Настраиваемые тайбрейки

Администратор турнира может задать порядок тайбрейков 2–4 на странице **Admin → Overview → Tiebreak order**. Доступные критерии:

| Идентификатор | Описание |
|---------------|----------|
| `buchholz` | Коэффициент Бухгольца — сумма очков соперников (подробнее в разделе 7) |
| `wins` | Количество побед (не считая BYE) |
| `head_to_head` | Результат личной встречи между участниками |

Если выбранный тайбрейк не разрешает ничью, система переходит к следующему критерию. Посевной номер является безусловным финальным разделителем — ничья в окончательной таблице невозможна.

### 6.2 Снимок таблицы

После закрытия каждого тура система сохраняет **снимок** турнирной таблицы. Это позволяет просматривать историческую таблицу на любой момент турнира на вкладке **Standings** публичной страницы.

## 7. Коэффициент Бухгольца

### 7.1 Определение

Коэффициент Бухгольца участника — это **сумма очков всех его соперников** по закрытым турам:

```
Buchholz(P) = Σ points(opponent_i)
```

где суммирование производится по всем реальным соперникам игрока P (партии, сыгранные за доской).

### 7.2 Исключения

- Партии **BYE** не учитываются при расчёте Бухгольца — ни очки самого BYE, ни «соперник» в этой партии.
- Партии с результатом **FF (Forfeit / технический проигрыш)** учитываются в истории встреч, однако не приносят очков ни одной стороне и тем самым снижают Бухгольц обоих игроков.
- Значение Бухгольца пересчитывается при каждом закрытии тура и отражает состояние **на момент закрытия**.

> FIDE Handbook C.02 (Tie-break regulations), ст. 13.1: «Buchholz score is the sum of the scores of each of the opponents of a player.»

### 7.3 Экономический смысл

Высокий коэффициент Бухгольца означает, что участник играл против сильных соперников (набравших много очков). При равных очках у двух участников выше ранжируется тот, чей «силовой уровень» противников был выше.
