## 1. Регистрация и вход

1. Перейдите на главную страницу и нажмите **Register**.
2. Введите email, имя и пароль, нажмите **Create account**.
3. При повторном посещении нажмите **Sign in**, введите email и пароль.
   - Если не хотите вводить пароль при каждом открытии браузера — отметьте **Remember me**.

---

## 2. Создание турнира

1. Перейдите в **Dashboard** (ссылка в шапке после входа).
2. Нажмите **New tournament**.
3. Заполните форму:

   | Поле | Описание |
   |------|----------|
   | **Name** | Название турнира (обязательно) |
   | **Description** | Произвольное описание (необязательно) |
   | **Time control** | Контроль времени, например `90+30` (текстовое поле) |
   | **Number of rounds** | Количество туров |
   | **Bye points** | Очки за технический выигрыш (обычно `1`) |
   | **Public** | Доступен ли турнир без авторизации |

4. Нажмите **Create**. Турнир создаётся со статусом **DRAFT**.

---

## 3. Регистрация участников

Участников можно добавлять пока турнир находится в статусе **DRAFT** или **OPEN**.

1. Откройте турнир → вкладка **Admin** → вкладка **Participants**.
2. В форме внизу введите **имя** участника и (опционально) его **рейтинг**.
3. Нажмите **Add**. Участник появится в списке.
4. Повторите для каждого участника.

**Редактирование и удаление:**
- Кнопка **Withdraw** переводит участника в статус «снят с соревнований» — он остаётся в истории, но не участвует в жеребьёвке.
- Кнопка **Remove** полностью удаляет участника (доступна до старта).

> **Совет.** Рейтинг влияет на начальный посев: участники с более высоким рейтингом получают меньший номер посева и в первом туре встречаются с игроками из нижней половины таблицы.

---

## 4. Открытие регистрации и запуск турнира

Турнир проходит через несколько статусов. Переход выполняется на вкладке **Admin → Overview**.

```
DRAFT → OPEN → RUNNING → FINISHED → ARCHIVED
```

### 4.1 DRAFT → OPEN
Нажмите **Open registration**. Турнир становится виден участникам (если отмечено «Public»), но жеребьёвка ещё не доступна. Можно продолжать добавлять участников.

> При необходимости можно вернуться обратно: **OPEN → DRAFT** кнопкой **Back to draft**.

### 4.2 OPEN → RUNNING
Когда все участники зарегистрированы, нажмите **Start tournament**.

При этом автоматически:
- каждому участнику присваивается **посевной номер** (seed) по убыванию рейтинга, при равном рейтинге — по порядку добавления;
- список участников фиксируется (добавить новых нельзя).

## 5. Проведение тура

### 5.1 Генерация жеребьёвки

1. Перейдите на вкладку **Admin → Pairings**.
2. Нажмите **Generate round N**.

   Система автоматически составит пары по швейцарской системе:
   - **Тур 1:** верхняя половина таблицы по посеву против нижней. Цвет первой пары определяется случайно, далее чередуется.
   - **Туры 2+:** игроки группируются по набранным очкам. Внутри группы приоритет отдаётся парам с дополнительными («комплементарными») цветами, чтобы минимизировать дисбаланс по цвету. Повторные встречи избегаются.
   - **Нечётное число участников:** технический выигрыш (BYE) получает последний по таблице игрок, ещё не имевший BYE.

   Жеребьёвка сохраняется как черновик — участники её ещё не видят.

### 5.2 Публикация тура

После проверки пар нажмите **Publish**. Тур переходит в статус **Published** и становится виден всем на публичной странице турнира.

---

## 6. Внесение результатов

1. На вкладке **Admin → Pairings** выберите нужный тур (кнопки с номерами туров вверху).
2. В столбце **Result** для каждой партии выберите результат из выпадающего списка:

   | Значение | Результат |
   |----------|-----------|
   | `1-0` | Победа белых |
   | `0-1` | Победа чёрных |
   | `½-½` | Ничья |
   | `FF` | Технический проигрыш (форфейт) |

3. Результат сохраняется автоматически при выборе.

> Партии с пометкой **BYE** закрыты для редактирования — результат проставляется автоматически.

---

## 7. Закрытие тура и турнирная таблица

Когда результаты всех партий внесены, нажмите **Close round**.

При закрытии:
- сохраняется **снимок турнирной таблицы** на момент окончания тура;
- пересчитываются **коэффициенты Бухгольца** и остальные показатели;
- тур переходит в статус **Closed** (отображается галочкой ✓).

Таблицу можно посмотреть на вкладках **Standings** и **Crosstable** публичной страницы турнира.

### Порядок сортировки в таблице

1. Очки (по убыванию)
2. Коэффициент Бухгольца (сумма очков соперников)
3. Количество побед
4. Личная встреча
5. Посевной номер (по возрастанию)

Порядок тайбрейков 2–4 настраивается на вкладке **Admin → Overview** в разделе **Tiebreak order**.

---

## 8. Повторение для следующих туров

Повторяйте шаги **5–7** для каждого тура:

```
Генерация → Публикация → Внесение результатов → Закрытие
```

Перейти к следующему туру можно только после закрытия предыдущего.

---

## 9. Завершение турнира

После закрытия **последнего тура** турнир автоматически переходит в статус **FINISHED**.

При необходимости его можно перевести в архив: **Admin → Overview → Archive**.

Финальная турнирная таблица доступна на публичной странице во вкладках **Standings** и **Crosstable**.

---

## 10. Публичный просмотр

Публичная страница турнира доступна по адресу `/tournaments/<slug>` и содержит четыре вкладки:

| Вкладка | Содержимое |
|---------|------------|
| **Info** | Название, описание, формат, статус |
| **Rounds** | Список туров, раскрываемые таблицы партий |
| **Standings** | Турнирная таблица (текущая или на конец выбранного тура) |
| **Crosstable** | Матрица результатов — результат и цвет каждого участника в каждом туре; клик по ячейке показывает подробности партии |

Ссылкой на публичную страницу можно поделиться с участниками и зрителями — авторизация для просмотра не требуется.
